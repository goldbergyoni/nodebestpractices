# Використовуйте .dockerignore для запобігання витоку секретів

<br/><br/>

### Пояснення за один абзац

Команда Docker build копіює локальні файли в середовище контексту збірки через віртуальну мережу. Будьте обережні — папки розробки та CI містять секрети, такі як .npmrc, .aws, .env файли та інші чутливі файли. Відповідно, Docker-образи можуть містити секрети та розкривати їх у небезпечних територіях (наприклад, Docker-репозиторій, сервери партнерів). У кращому світі Dockerfile повинен явно вказувати, що копіюється. На додаток до цього включіть файл .dockerignore, який діє як остання мережа безпеки, що фільтрує непотрібні папки та потенційні секрети. Це також прискорює збірку — залишаючи поза увагою загальні папки розробки, які не мають користі в продакшені (наприклад, .git, результати тестів, конфігурація IDE), збирач може краще використовувати кеш та досягти кращої продуктивності

<br/><br/>

### Приклад коду – Хороший .dockerignore за замовчуванням для Node.js

<details>
<summary><strong>.dockerignore</strong></summary>

```
**/node_modules/
**/.git
**/README.md
**/LICENSE
**/.vscode
**/npm-debug.log
**/coverage
**/.env
**/.editorconfig
**/.aws
**/dist
```

</details>

### Приклад коду Антипатерн – Рекурсивне копіювання всіх файлів

<details>
<summary><strong>Dockerfile</strong></summary>

```dockerfile
FROM node:12-slim AS build

WORKDIR /usr/src/app
# Наступний рядок копіює все
COPY . .

# Решта йде тут

```

</details>
