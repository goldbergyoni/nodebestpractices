# Очищайте кеш NODE_MODULE

<br/><br/>

### Пояснення за один абзац

Менеджери пакетів Node, npm та Yarn, кешують встановлені пакети локально, щоб майбутні проекти, яким потрібні ті самі бібліотеки, не потребували завантаження з віддаленого репозиторію. Хоча це дублює пакети та споживає більше сховища — це окупається в локальному середовищі розробки, яке зазвичай продовжує встановлювати ті самі пакети. У Docker-контейнері це збільшення сховища марне, оскільки він встановлює залежність лише один раз. Видаляючи цей кеш одним рядком коду, десятки МБ зрізаються з образу. Роблячи це, переконайтеся, що він не виходить з ненульовим кодом і не провалює CI-збірку через проблеми з кешуванням — цього можна уникнути, включивши прапорець --force.

*Зверніть увагу, що це не актуально, якщо ви використовуєте багатоетапну збірку, поки ви не встановлюєте нові пакети на останньому етапі*
