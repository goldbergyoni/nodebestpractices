# Дозвольте оркестратору Docker перезапускати та реплікувати процеси

<br/><br/>

### Пояснення за один абзац

Оркестратори Docker runtime, такі як Kubernetes, дійсно добре приймають рішення щодо здоров'я та розміщення контейнерів: Вони подбають про максимізацію кількості контейнерів, балансування їх між зонами та враховують багато факторів кластера при прийнятті цих рішень. Само собою зрозуміло, вони ідентифікують процеси, що виходять з ладу (тобто контейнери), і перезапускають їх у правильному місці. Незважаючи на це, деякі можуть спокуситися використовувати кастомний код або інструменти для реплікації Node-процесу для утилізації CPU або перезапуску процесу при збої (наприклад, модуль Cluster, PM2). Ці локальні інструменти не мають перспективи та даних, доступних на рівні кластера. Наприклад, коли ресурси екземплярів можуть розмістити 3 контейнери і враховуючи 2 регіони або зони, Kubernetes подбає про розподіл контейнерів між зонами. Таким чином, у випадку зонального або регіонального збою застосунок залишиться живим. З іншого боку, при використанні локальних інструментів для перезапуску процесу оркестратор Docker не знає про помилки і не може приймати продумані рішення, такі як переміщення контейнера на новий екземпляр або зону.
