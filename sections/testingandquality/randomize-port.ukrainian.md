# Вказуйте порт у продакшені, рандомізуйте при тестуванні

<br/><br/>

### Пояснення в один абзац

При написанні компонентних/інтеграційних тестів веб-сервер має запускатися тестами в тому ж процесі — це відкриває двері для багатьох бажаних можливостей тестування, таких як mocking, покриття коду тощо. У багатопроцесному тест-раннері буде відкрито кілька екземплярів веб-сервера. Якщо ці екземпляри спробують відкрити той самий порт, вони зіткнуться. Лише при тестуванні дозвольте серверу рандомізувати порт для запобігання колізій. Цього легко досягти, вказавши [ефемерний порт](https://uk.wikipedia.org/wiki/Ефемерний_порт), число нуль, щоб операційна система виділила доступний порт.

<br/><br/>

### Приклад коду – запуск веб-сервера з урахуванням тестування

```javascript
// api-under-test.js
const initializeWebServer = async () => {
  return new Promise((resolve, reject) => {
    // Фіксований порт у продакшені, нульовий порт (ефемерний) для тестування
    const webServerPort = process.env.PORT ? process.env.PORT : 0;
    expressApp = express();
    connection = expressApp.listen(webServerPort, () => {
      // Без порту
      resolve(expressApp);
    });
  });
};

// test.js
beforeAll(async () => {
  expressApp = await initializeWebServer(); // Без порту
});
```
