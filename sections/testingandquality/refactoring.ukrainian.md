# Рефакторинг

<br/><br/>

### Пояснення за один абзац

Рефакторинг є важливим процесом в ітеративному потоці розробки. Видалення "Code Smells" (поганих практик кодування), таких як дубльований код, довгі методи, довгий список параметрів, покращить ваш код і зробить його більш підтримуваним. Використання інструментів статичного аналізу допоможе вам знайти ці code smells і побудувати процес навколо рефакторингу. Додавання цих інструментів до вашої CI збірки допоможе автоматизувати процес перевірки якості. Якщо ваш CI інтегрується з інструментом, таким як Sonar або Code Climate, збірка завершиться невдачею, якщо виявить code smells, і повідомить автора про те, як вирішити проблему. Ці інструменти статичного аналізу доповнюють інструменти лінтингу, такі як ESLint. Більшість інструментів лінтингу зосереджуються на стилях коду, таких як відступи та відсутні крапки з комою (хоча деякі знаходять code smells, такі як довгі функції) в одному файлі, тоді як інструменти статичного аналізу зосереджуються на пошуку code smells (дубльований код, аналіз складності тощо), які є в окремих файлах і кількох файлах.
