# Уникайте завантаження модулів за допомогою змінної

### Пояснення за один абзац

Уникайте require/import іншого файлу зі шляхом, який був переданий як параметр, через побоювання, що він міг походити з користувацького вводу. Це правило можна розширити на доступ до файлів загалом (тобто `fs.readFile()`) або інших конфіденційних ресурсів з динамічними змінними, що походять з користувацького вводу.

### Приклад коду

```javascript
// небезпечно, оскільки змінна helperPath може бути змінена користувацьким вводом
const badWayToRequireUploadHelpers = require(helperPath);

// безпечно
const uploadHelpers = require('./helpers/upload');
```
