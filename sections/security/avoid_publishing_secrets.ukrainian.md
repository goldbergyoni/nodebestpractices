# Уникайте публікації секретів до npm реєстру

### Пояснення за один абзац
Слід вживати запобіжних заходів, щоб уникнути ризику випадкової публікації секретів до публічних npm реєстрів. Файл `.npmignore` можна використовувати для внесення до чорного списку конкретних файлів або папок, або масив `files` у `package.json` може діяти як білий список.

Щоб отримати уявлення про те, що npm publish насправді опублікує до реєстру, можна додати прапорець `--dry-run` до команди npm publish, щоб отримати детальний огляд створеного tarball-пакета.

Важливо зазначити, що якщо проект використовує як `.npmignore`, так і `.gitignore` файли, все, що не є в `.npmignore`, публікується до реєстру (тобто файл `.npmignore` перевизначає `.gitignore`). Ця умова є поширеним джерелом плутанини і є проблемою, яка може призвести до витоку секретів. Розробники можуть оновити файл `.gitignore`, але забути оновити `.npmignore` також, що може призвести до того, що потенційно чутливий файл не буде відправлено до системи контролю версій, але все одно буде включено до npm пакета.

### Приклад коду
Приклад файлу .npmignore
```
# Tests
test
coverage

# Build tools
.travis.yml
.jenkins.yml

# Environment
.env
.config

```

Приклад використання масиву files у package.json

```json
{ 
  "files" : [
    "dist/moment.js",
    "dist/moment.min.js"
  ]
}
```

### Що кажуть інші блогери

З блогу [Liran Tal & Juan Picado at Snyk](https://snyk.io/blog/ten-npm-security-best-practices/):
> ... Іншою хорошою практикою є використання властивості files у package.json, яка працює як білий список і визначає масив файлів, які мають бути включені до пакета, що створюється та встановлюється (тоді як файл ignore функціонує як чорний список). Властивість files і файл ignore можуть використовуватися разом, щоб визначити, які файли мають бути явно включені, а також виключені з пакета. При використанні обох, перша властивість files у package.json має пріоритет над файлом ignore.

З [npm блогу](https://blog.npmjs.org/post/165769683050/publishing-what-you-mean-to-publish)
> ... Коли ви запускаєте npm publish, npm пакує всі файли в поточному каталозі. Він приймає деякі рішення за вас про те, що включити і що ігнорувати. Для прийняття цих рішень він використовує вміст кількох файлів у каталозі вашого проекту. Ці файли включають .gitignore, .npmignore і масив files у package.json. Він також завжди включає певні файли і ігнорує інші.
