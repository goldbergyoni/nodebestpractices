# Уникайте JavaScript eval виразів

### Пояснення за один абзац

`eval()`, `setTimeout()`, `setInterval()` та `new Function()` є глобальними функціями, які часто використовуються в Node.js і приймають рядковий параметр, що представляє JavaScript вираз, оператор або послідовність операторів. Проблема безпеки використання цих функцій полягає в можливості того, що ненадійний користувацький ввід може потрапити до виконання коду, що призведе до компрометації сервера, оскільки виконання користувацького коду по суті дозволяє зловмиснику виконувати будь-які дії, які можете виконати ви. Рекомендується рефакторити код, щоб не покладатися на використання цих функцій, де користувацький ввід може бути переданий до функції та виконаний.

### Приклад коду

```javascript
// приклад шкідливого коду, який зловмисник зміг ввести
const userInput = "require('child_process').spawn('rm', ['-rf', '/'])";

// шкідливий код виконано
eval(userInput);
```

### Що кажуть інші блогери

З книги Essential Node.js Security від [Liran Tal](https://leanpub.com/nodejssecurity):
> Функція eval() є, мабуть, однією з найбільш засуджуваних частин JavaScript з точки зору безпеки. Вона парсить рядок JavaScript як текст і виконує його так, ніби це був JavaScript код. Змішування цього з ненадійним користувацьким вводом, який може потрапити до eval() — це рецепт катастрофи, яка може закінчитися компрометацією сервера.
