# Приховуйте деталі помилок від клієнта

### Пояснення за один абзац

Розкриття деталей помилок додатку клієнту в продакшені слід уникати через ризик розкриття конфіденційних деталей додатку, таких як шляхи файлів сервера, сторонні модулі, що використовуються, та інші внутрішні робочі процеси додатку, які можуть бути використані зловмисником.
Express постачається з вбудованим обробником помилок, який обробляє будь-які помилки, що можуть виникнути в додатку. Ця функція обробки помилок за замовчуванням додається в кінець стеку функцій middleware.
Якщо ви передаєте помилку до `next()` і не обробляєте її в користувацькому обробнику помилок, вона буде оброблена вбудованим обробником помилок Express; помилка буде записана клієнту зі стек-трейсом. Ця поведінка буде дійсною, коли `NODE_ENV` встановлено в `development`, однак коли `NODE_ENV` встановлено в `production`, стек-трейс не записується, лише код HTTP відповіді.

### Приклад коду: Обробник помилок Express

```javascript
// обробник помилок для продакшену
// стек-трейси не витікають до користувача
app.use((err, req, res, next) => {
    res.status(err.status || 500);
    res.render('error', {
        message: err.message,
        error: {}
    });
});
```
