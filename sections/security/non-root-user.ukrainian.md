# Запускайте Node.js як не-root користувач

### Пояснення за один абзац

Згідно з 'Принципом найменших привілеїв' користувач/процес повинен мати доступ лише до необхідної інформації та ресурсів. Надання root доступу зловмиснику відкриває цілий новий світ шкідливих ідей, таких як маршрутизація трафіку на інші сервери. На практиці більшість додатків Node.js не потребують root доступу і не запускаються з такими привілеями. Однак є два поширених сценарії, які можуть підштовхнути до використання root:

- для отримання доступу до привілейованого порту (наприклад, порт 80) Node.js повинен запускатися як root
- Docker контейнери за замовчуванням запускаються як root(!). Рекомендується, щоб веб-додатки Node.js слухали непривілейовані порти і покладалися на зворотний проксі, такий як nginx, для перенаправлення вхідного трафіку з порту 80 до вашого Node.js додатку. При створенні Docker образу високозахищені додатки повинні запускати контейнер з альтернативним не-root користувачем. Більшість Docker кластерів (наприклад, Swarm, Kubernetes) дозволяють декларативно встановлювати контекст безпеки

### Приклад коду - Створення Docker образу як не-root

```dockerfile
FROM node:latest

COPY package.json .
RUN npm install
COPY . .
EXPOSE 3000
USER node
CMD ["node", "server.js"]
```

### Цитата з блогу: "За замовчуванням Docker запускає контейнер як root"

З репозиторію docker-node від [eyalzek](https://github.com/nodejs/docker-node/blob/master/docs/BestPractices.md#non-root-user):
> За замовчуванням Docker запускає контейнер як root, що всередині контейнера може становити проблему безпеки. Ви хотіли б запускати контейнер як непривілейований користувач, де це можливо. Node образи надають користувача node для цієї мети. Docker образ потім можна запустити з користувачем node наступним чином: "-u 'node'"

<br/><br/>

### Цитата з блогу: "Зловмисник матиме повний контроль над вашою машиною"

З блогу Don't run Node.js as root від [Olivier Lalonde](http://syskall.com/dont-run-node-dot-js-as-root/):
> Дійсно, якщо ви запускаєте свій сервер як root і він буде зламаний через вразливість у вашому коді, зловмисник матиме повний контроль над вашою машиною. Це означає, що зловмисник потенційно може стерти весь ваш диск або ще гірше. З іншого боку, якщо ваш сервер працює з правами звичайного користувача, зловмисник буде обмежений цими правами.

<br/><br/>

### Цитата з блогу: "Якщо вам потрібно запустити додаток на порту 80 або 443, ви можете зробити перенаправлення портів"

З блогу Developing Secure Node.js Applications — A Broad Guide від [Deepal Jayasekara](https://jsblog.insiderattack.net/developing-secure-node-js-applications-a-broad-guide-286afdec69ce):
> Ніколи не запускайте Node.js як root. Запуск node.js як root зробить ситуацію гіршою, якщо зловмисник якось отримає контроль над вашим додатком. У цьому сценарії зловмисник також отримає права root, що може призвести до катастрофи. Якщо вам потрібно запустити додаток на порту 80 або 443, ви можете зробити перенаправлення портів за допомогою iptables або розмістити фронт-енд проксі, такий як nginx або apache, який маршрутизує запити з порту 80 або 443 до вашого додатку
