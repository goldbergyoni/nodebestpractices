# Запобігайте небезпечним перенаправленням

### Пояснення за один абзац

При реалізації перенаправлень у Node.js та/або Express важливо виконувати валідацію вводу на стороні сервера.
Якщо зловмисник виявить, що ви не валідуєте зовнішній, наданий користувачем ввід, він може використати цю вразливість, публікуючи спеціально створені посилання на форумах, соціальних мережах та інших публічних місцях, щоб змусити користувачів натиснути на них.

Приклад: Небезпечне express перенаправлення з використанням користувацького вводу
```javascript
const express = require('express');
const app = express();

app.get('/login', (req, res, next) => {

  if (req.session.isAuthenticated()) {
    res.redirect(req.query.url);
  }

}); 
```

Запропоноване виправлення для уникнення небезпечних перенаправлень — не покладатися на користувацький ввід. Якщо користувацький ввід повинен використовуватися, можна використовувати білі списки безпечних перенаправлень, щоб уникнути розкриття вразливості.

Приклад: Білий список безпечних перенаправлень
```javascript
const whitelist = { 
  'https://google.com': 1 
};

function getValidRedirect(url) { 
    // перевіряємо, чи url починається з одного слеша 
  if (url.match(/^\/(?!\/)/)) { 
    // Додаємо наш домен на початок, щоб переконатися 
    return 'https://example.com' + url; 
  } 
    // В іншому випадку перевіряємо за білим списком
  return whitelist[url] ? url : '/'; 
}

app.get('/login', (req, res, next) => {

  if (req.session.isAuthenticated()) {
    res.redirect(getValidRedirect(req.query.url));
  }

}); 
```


### Що кажуть інші блогери

З блогу [NodeSwat](https://blog.nodeswat.com/unvalidated-redirects-b0a2885720db):
> На щастя, методи пом'якшення цієї вразливості досить прямолінійні — не використовуйте невалідований користувацький ввід як основу для перенаправлення.

З блогу [Hailstone](https://blog.hailstone.io/how-to-prevent-unsafe-redirects-in-node-js/)
> Однак, якщо серверна логіка перенаправлення не валідує дані, що надходять у параметр url, ваші користувачі можуть опинитися на сайті, який виглядає точно як ваш (examp1e.com), але в кінцевому підсумку служить потребам кримінальних хакерів!
