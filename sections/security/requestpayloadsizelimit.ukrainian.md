# Обмежуйте розмір payload за допомогою зворотного проксі або middleware

### Пояснення за один абзац

Парсинг тіл запитів, наприклад, JSON-кодованих payload, є операцією, що вимагає значних ресурсів продуктивності, особливо при більших запитах.
При обробці вхідних запитів у вашому веб-додатку слід обмежувати розмір їхніх відповідних payload. Вхідні запити з
необмеженим розміром тіла/payload можуть призвести до поганої роботи вашого додатку або збою через відмову в обслуговуванні або інших небажаних побічних ефектів.
Багато популярних middleware-рішень для парсингу тіл запитів, такі як вже включений пакет `body-parser` для express, надають
опції для обмеження розмірів payload запитів, що полегшує розробникам реалізацію цієї функціональності. Ви також можете
інтегрувати обмеження розміру тіла запиту у ваше програмне забезпечення зворотного проксі/веб-сервера, якщо це підтримується. Нижче наведені приклади обмеження розмірів запитів за допомогою
`express` та/або `nginx`.

### Приклад коду для `express`

```javascript
const express = require('express');

const app = express();

app.use(express.json({ limit: '300kb' })); // body-parser за замовчуванням має обмеження розміру тіла 100kb

// Запит з json тілом
app.post('/json', (req, res) => {

    // Перевіряємо, чи content-type payload запиту відповідає json, оскільки body-parser не перевіряє типи вмісту
    if (!req.is('json')) {
        return res.sendStatus(415); // -> Unsupported media type, якщо запит не має JSON тіла
    }

    res.send('Ура, це спрацювало!');
});

app.listen(3000, () => console.log('Приклад додатку слухає на порту 3000!'));
```

🔗 [**Документація Express для express.json()**](http://expressjs.com/en/4x/api.html#express.json)

### Приклад конфігурації для `nginx`

```nginx
http {
    ...
    # Обмежити розмір тіла для ВСІХ вхідних запитів до 1 MB
    client_max_body_size 1m;
}

server {
    ...
    # Обмежити розмір тіла для вхідних запитів до цього конкретного серверного блоку до 1 MB
    client_max_body_size 1m;
}

location /upload {
    ...
    # Обмежити розмір тіла для вхідних запитів до цього маршруту до 1 MB
    client_max_body_size 1m;
}
```

🔗 [**Документація Nginx для client_max_body_size**](http://nginx.org/en/docs/http/ngx_http_core_module.html#client_max_body_size)
