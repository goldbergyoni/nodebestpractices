# Змініть налаштування сесійного middleware за замовчуванням

<br/><br/>


### Пояснення за один абзац

Багато популярних сесійних middleware не застосовують найкращі практики/безпечні налаштування cookies з коробки. Налаштування цих параметрів від значень за замовчуванням пропонує більший захист як для користувача, так і для додатку, зменшуючи загрозу атак, таких як викрадення сесії та ідентифікація сесії.

Найпоширеніше налаштування, залишене за замовчуванням — це `name` сесії — у `express-session` це `connect.sid`. Зловмисник може використовувати цю інформацію для ідентифікації базового фреймворку веб-додатку, а також специфічних для модуля вразливостей. Зміна цього значення на щось інше, ніж за замовчуванням, ускладнить визначення того, який сесійний механізм використовується.

Також у `express-session` опція `cookie.secure` встановлена в false як значення за замовчуванням. Зміна цього на true обмежить передачу cookie лише через https, що забезпечить безпеку від атак типу man-in-the-middle

<br/><br/>


### Приклад коду: Встановлення безпечних налаштувань cookie

```javascript
// використання express session middleware
app.use(session({  
  secret: 'youruniquesecret', // секретний рядок, що використовується при підписі ID сесії, який зберігається в cookie
  name: 'youruniquename', // встановіть унікальне ім'я, щоб прибрати connect.sid за замовчуванням
  cookie: {
    httpOnly: true, // мінімізувати ризик XSS атак, обмежуючи клієнта від читання cookie
    secure: true, // надсилати cookie лише через https
    maxAge: 60000*60*24 // встановити термін дії cookie в мс
  }
}));
```

### Що кажуть інші блогери

З [блогу NodeSource](http://nodesource.com/blog/nine-security-tips-to-keep-express-from-getting-pwned/): 
> ...Express має налаштування cookie за замовчуванням, які не є високобезпечними. Їх можна вручну посилити для підвищення безпеки — як для додатку, так і для його користувача.*
