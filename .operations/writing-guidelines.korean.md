# 우리의 콘텐츠 작성 원칙

방문자들의 읽기와 학습 경험을 어떻게 향상시키는가.

## 1. 단순함이 더 나은 것보다 낫다.

(더 나은 것을 추구하는 과정에서 복잡성이 증가하는 순간, "단순함"이 오히려 더 큰 가치를 가진다.)

읽기 쉽고 지식을 쉽게 흡수할 수 있게 만드는 것이 우리의 미션이며, 우리는 이를 위해 콘텐츠를 선별하고 다듬습니다.
그래서 우리는 복잡하고 부담스러운 주제는 단순한 리스트 형태로 전환하고, 과도하게 많은 정보는 간단하고 핵심적인 내용으로 바꾸며, 논란을 일으키기 쉬운 민감한 주제는 피합니다. 또한 주관적인 의견보다 일반적으로 인정된 관행을 우선합니다.

## 2. 근거 기반이며 신뢰할 수 있어야 한다.

독자들은 우리가 제공하는 콘텐츠의 신뢰성에 대해 확신할 수 있어야 합니다. 이를 위해 해당 주제에 대한 참고문헌, 데이터 및 기타 자료와 같은 증거를 포함시킵니다. 실질적으로, 신뢰할 수 있는 출처의 인용문을 포함시키고, 벤치마크를 제시하며, 관련 디자인 패턴이나 주장을 입증할 수 있는 과학적 측정 기준을 제시할 것을 지향합니다.

## 3. MECE 원칙 (상호 배타적이며 포괄적)

콘텐츠가 잘 정리되고 신뢰성을 갖추는 것뿐 아니라, 빠르게 훑어보아도 주제가 빠짐없이 다뤄졌다고 느껴져야 합니다. 중요한 하위 주제는 절대로 누락되어서는 안 됩니다.

## 4. 일관된 포맷

모든 콘텐츠는 고정된 템플릿을 사용해 작성합니다.
새로운 항목을 추가할 때는 기존 항목의 형식을 그대로 복사해 확장해야 합니다.
자세한 내용은 [이 템플릿](../sections/template.md)을 참고하세요.

## 5. 주제는 항상 Node.js다.

각 조언은 일반적인 소프트웨어 개발이 아닌 Node.js와 직접적으로 관련되어야 합니다. Node.js에서 일반적인 패턴/규칙을 구현하라고 조언할 때에도 Node.js 관점으로 설명해야 합니다.
예를 들어, "보안상의 이유로 모든 요청 입력을 검증하라"고 조언할 때는 Node.js 문맥에 맞게 "미들웨어를 사용하여 요청 입력을 검증하세요"와 같은 방식으로 표현합니다.
Node.js에 특화되지 않고 모든 언어에서 동일하게 적용되는 항목(예: Python 및 Java에서 동일하게 보이는 경우)은 일반적인 컨테이너 항목 내에 포함해야 합니다. (예: 항목 6.5 참고).

## 6. 상위 도구만 다룹니다.

때로는 특정 문제를 해결하는 데 도움이 되는 npm 패키지, 오픈소스 도구, 또는 상용 솔루션을 소개하는 것이 유용할 때가 있습니다.
하지만 너무 많은 추천을 제공하거나 신뢰하기 어려운 프로젝트를 다루지 않게 하기 위해 다음 기준을 마련했습니다.

- 추천은 상위 3개 도구만 포함합니다.
  - 검색 엔진(구글) 또는 GitHub(인기순 정렬)에서 관련 키워드로 검색했을 때 상위 3위 안에 등장하는 도구만 추천 목록에 포함할 수 있습니다.
- npm 패키지의 경우 평균적으로 하루에 최소 750회 이상 다운로드되어야 합니다.
- 오픈소스 프로젝트라면 최근 6개월 내에 최소 1회 이상 업데이트된 프로젝트만 포함합니다.
